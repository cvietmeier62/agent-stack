# Project Generator

Orchestrator skill that produces a complete project spec — the same 7-file system Christian uses for every project. Each file is generated by a dedicated sub-step with its own instructions.

## What Gets Generated

```
{project-name}/
├── CLAUDE.md                          # Steering index (50-60 lines)
├── .claude/
│   ├── rules/
│   │   ├── stack.md                   # Technology decisions (locked)
│   │   ├── conventions.md             # Coding patterns + UI rules
│   │   └── architecture.md            # System design + data flow
│   └── skills/
│       └── panel-discussion/SKILL.md  # Portable thinking tool (copied)
├── docs/
│   ├── DESIGN.md                      # Component specs per section
│   ├── TASKS.md                       # Ordered tasks with acceptance criteria
│   └── PRD.md                         # Product requirements (optional, for larger projects)
└── .gitignore
```

## Orchestration

Run the sub-steps IN ORDER. Each produces one or more files. Pass context forward.

```
Step 1: DISCOVER  →  answers to 5 questions  →  discovery.json (in memory, not written)
Step 2: RESEARCH  →  verified context         →  feeds into all subsequent steps
Step 3: STACK     →  .claude/rules/stack.md
Step 4: ARCH      →  .claude/rules/architecture.md
Step 5: CONV      →  .claude/rules/conventions.md
Step 6: DESIGN    →  docs/DESIGN.md
Step 7: TASKS     →  docs/TASKS.md
Step 8: STEERING  →  CLAUDE.md (reads all rules + design to generate the index)
Step 9: FINALIZE  →  .gitignore, copy skills, git init, git push
```

Each sub-step has instructions in its own reference file. Read `references/step-{N}.md` for the detailed instructions for that step.

## How to Run

1. Read `references/step-1-discover.md` — ask Christian the questions
2. Based on answers, read `references/step-2-research.md` — identify gaps, fill them
3. Read each subsequent step file and execute it
4. After Step 9, present the GitHub URL to Christian
