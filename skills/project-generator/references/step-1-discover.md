# Step 1: Discover

Ask Christian up to 5 questions. Stop as soon as you have enough to make decisions. If his initial description already answers most questions, confirm and move on — don't interrogate.

## Questions (ask only what's not already clear)

1. **What are you building?** — one sentence
2. **Who uses it?** — persona + workflow
3. **What's the main interface?** — chat, dashboard, IDE-style, form, API
4. **What data does it work with?** — what goes in, what comes out
5. **Where does it deploy?** — Vercel (default) or AWS? What MCPs/infra exist?

## Rules

- If Christian gives a rich description that covers 3+ questions, don't re-ask. Summarize what you understood and confirm.
- Use Christian's defaults for anything not mentioned (see SKILL.md defaults section).
- Capture the answers mentally — they feed every subsequent step.
- Output: a brief summary of the project (2-3 sentences) that Christian confirms before proceeding.

## Example

Christian: "I want a chatbot demo built on Strands for AWS"

You already know: chat interface, Strands SDK, AWS deploy, dark mode, AI Elements. You might only ask: "Any specific features beyond basic chat, or is this a minimal demo?" — one question, then move on.
